AREA 1: MODULO B (Propagacao D+1)
==================================

Status: NAO INICIADO
Impacto: CRITICO - Necessario para dissertacao completa
Prioridade: MAXIMA

Problema:
- Modulo A detecta fogos reais vs espurios (COMPLETO)
- Modulo B prediz propagacao D+1 (AUSENTE)
- Dissertacao requer AMBOS modulos

Solucao Proposta:

Objetivo: Prever ONDE fogo vai se espalhar amanha

Dataset:
- Grid regular 250-500m sobre MATOPIBA
- Label: Estava queimado HOJE (MCD64A1) ou nao?
- Dados: 2020-2023 (3 anos = 1000+ casos)

Features:
1. Meteorologicas (ERA5):
   - Temperatura
   - Umidade relativa
   - Velocidade vento (direcao + magnitude)
   - Precipitacao

2. Topograficas:
   - Elevacao (DEM/SRTM)
   - Inclinacao

3. Vegetacao (Sentinel-2):
   - NDVI (saude da vegetacao)
   - Tipo de cobertura

4. Historicas:
   - Queimado anteriormente? (365 dias)
   - Densidade hotspots vizinhanca
   - Proximidade fogo ativo hoje

Modelo:
- Algoritmo: LightGBM (consistente com Modulo A)
- Validacao: Temporal (treina com 2020-2022, testa 2023)
- Metricas: IoU (Intersection over Union), Brier Score, AUC-ROC
- Performance esperada: IoU ~0.65-0.75

Output:
- Probabilidade propagacao (0-1) para cada celula grid
- Mapa GeoTIFF com risco D+1
- Shapefile com areas alto risco

Timeline: 2-3 semanas (semanas 2-3)

Dependencias:
- Dados ERA5 (ja disponivel)
- MCD64A1 com datas exatas (precisa validacao)
- Grid criacao (novo codigo)

Documentacao:
- MODULO_B_PROPAGACAO.md (~400 linhas)
- Scripts: src/models/train_module_b.py, predict_module_b.py
- Notebook: demo_modulo_b.ipynb

AREA 2: PIPELINE INTEGRADO (A + B)
====================================

Status: NAO EXISTE
Impacto: ALTA - Necessario para producao final
Prioridade: ALTA (apos Modulo B)

Problema:
- Modulo A prediz (real vs espurio)
- Modulo B prediz (propagacao D+1)
- Faltam conectar em pipeline unico

Solucao Proposta:

Script: run_integrated_pipeline.py

Fluxo:
1. INPUT: CSV hotspots FIRMS atuais
   - latitude, longitude, confidence, data

2. MODULO A: Classificacao
   - Carregar modelo treinado
   - Extrair features automaticamente
   - Predizer: real vs espurio + probabilidade

3. MODULO B: Propagacao
   - Filtrar apenas hotspots REAIS (Modulo A)
   - Criar grid ao redor (buffer 50km)
   - Extrair features meteorologicas+topograficas
   - Predizer: propagacao D+1 + risco

4. INTEGRACAO:
   - Combinar resultados
   - Mapa final: Classificacao + Risco propagacao
   - Recomendacoes: Prioridade intervencao

5. OUTPUT:
   - CSV: Predicoes detalhadas
   - GeoJSON: Mapa interativo
   - PDF: Relatorio automatico
   - JSON: Metadados e metricas

Interface:
- CLI: python run_integrated_pipeline.py --input hotspots.csv
- Opcional: REST API (Flask/FastAPI)
- Opcional: Web UI (Streamlit)

Timeline: 1 semana (semana 4)

Beneficio:
- Sistema end-to-end
- Pronto para producao real
- Pronto para publicacao academica

AREA 4: FEATURES ADICIONAIS PARA MODULO A
===========================================

Status: PARCIAL (20 features atuais)
Impacto: MEDIA - Pode melhorar +2-3% acuracia
Prioridade: MEDIA (paralelo com validacao, semana 1)

Features Candidatas:

1. SPI (Standardized Precipitation Index)
   - Mede anomalia de precipitacao
   - Positivo: -1 = muito seco, +1 = muito umido
   - Fonte: ERA5
   - Esperado: Fogo mais comum em SPI negativo
   - Potencial: +1% acuracia

2. Tipo de Bioma/Vegetacao
   - Classificacao: Cerrado, Floresta, Transicao
   - Fonte: MapBiomas ou Sentinel-2 classification
   - Esperado: Padroes fogo variam por bioma
   - Potencial: +1% acuracia

3. Anomalia de Temperatura
   - Temperatura - Media Historica (30 anos)
   - Fonte: ERA5
   - Esperado: Temp acima media = mais fogo
   - Potencial: +0.5% acuracia

4. Razao Vento/Umidade
   - Combinacao: (Vento * 10) / Umidade
   - Esperado: Indica condicoes favoraveis fogo
   - Potencial: +0.5% acuracia

Validacao:
- Adicionar features uma por uma
- Treinar novo modelo
- Comparar: Acuracia, ROC-AUC, PR-AUC
- Verificar importancia via SHAP

Implementacao:
- Script: add_features.py
- Atualizar: feature_engineering.py
- Retreinar: train_module_a.py com novo set

Timeline: 2-3 dias (paralelo semana 1)

Risco: Overfitting (validar com cross-validation)
